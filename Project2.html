Introduction

  • This tutorial walks you through the steps of generating a flask application and deploying it to an AWS Elastic Beanstalk environment.

  • It is classified as a microframework since it does not enforce the user to choose tools and dependencies. it allows the user to choose the database and template engine depends on the project requirement. Flask is commonly used to create a simple web application but complexity can be extensible with the help of third party libraries

  • In this project we introduced "Flask-SQLAlchemy" which is an extension for Flask that adds support for SQLAlchemy to the application. It aims to create a communication between python programming to database.

  • Jinja is a template engine written in python comes along with flask. This template engine helps us to keep the view code like HTML,CSS and JS organized.

  • AWS Elastic Beanstalk is an orchestration service offered by Amazon Web Services for deploying applications which orchestrates various AWS services, including EC2, S3
